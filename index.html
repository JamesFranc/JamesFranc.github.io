Who am I?

I enjoy making tools that people like using, but tinkering to see if I can make something work is a passion, of sorts.

A few years back after switching from chemistry to computer science, I began programming and haven't looked back. This page is a return to the basics project for me using only HTML, CSS and JS.

Why make this?

My goal for the project was to solidify knowledge of these foundational elements for web based software. Designing an attractive site using the basics took some time, and a few iterations to get where it is. Responsiveness and mobile compatibility were key objectives in working on the project. Along the way I got to learn about blocking JS and CSS, server-side configuration, browser caching, image optimization, Google's PageInsights and how mobile and desktop pages score.

In previous projects I've used AWS and their myriad of wonderful services. For exposure and learning's sake I chose to use Digital Ocean, and have been very pleased with it's ease of use and functionality.

My aim is to create a responsive, clean website that could be used to display my technical abilities and any knowledge gained in the process of making.

What do I want to do?

Whether it's creating a new form in a modal, implementing error checking inside of a field via JS, adding business logic, standing up backend services, designing user interaction or planning the infrastructure; I enjoy it all. Full-stack development gets me excited because seeing the big picture is important. Building tools from a full-stack perspective helps me understand the how and why, which makes for a more informed decision making process.

I enjoy learning new skills. Using new languages, frameworks and libraries in order to engage users and add value to projects is both fun and challenging. If there's a particular technology needed to solve a problem, I will learn it.
